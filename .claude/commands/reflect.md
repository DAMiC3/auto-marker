Reflect honestly on the work just completed in this session. Optional focus: $ARGUMENTS

Be candid, not flattering. The point is to get better over time, so surface real friction, not a victory lap.

## Produce three sections

1. **What went right** — what worked, decisions that paid off, things to keep doing
2. **What went wrong** — mistakes, wrong turns, wasted effort, misunderstandings, anything that needed redoing. Include process friction (e.g. acting before confirming, missing context)
3. **How to do it better next time** — concrete, actionable changes. Not "be more careful" — specifics like "check X before Y", "ask before deleting Z"

## Then log it (this is what makes it a loop)

Append a dated entry to `docs/reflections.md` (create the file with an `# AutoMark — reflections log` heading if it doesn't exist). Format each entry:

```
## <YYYY-MM-DD> — <short title of the work>
**Right:** ...
**Wrong:** ...
**Do better:** ...
```

Keep entries tight — bullet points, no padding. This file is the accumulating memory of lessons; future sessions can read it to avoid repeating mistakes. If a lesson is important and durable enough to change default behaviour, also tell the user it may be worth adding to `AGENTS.md`.
